ネットワーク実習課題

R1(JSP)

必修

  • (A1-1) 入力画面でイベント名とその年月日を入力し，ボタンを押すと，「XX（イベント名）まであとYY日です．」と表示されるページを作れ．（例：関西万博[2025年4月13日]まであとZZ日．）

  • (A1-2) 外貨から円へのカレンシーコンバータ（例えば，https://www.xe.com/ucc/ で右側を円に固定したようなもの）のページを作れ．両替レートはプログラム中に書き込んでおいても構わない．外貨の数量と種別を入力して「両替」ボタンを押すと，金額が表示される．少なくとも三つの通貨に関して両替できること．外貨の選択にはラジオボタン，チェックボックス，選択ボックス，リストボックスのいずれかを利用すること．

オプション

  • (B1-1) 年と月を入力すると，その月のカレンダーを表示するプログラムを作れ．また日曜日は赤字，土曜日は青字で表示すること．またカレンダーに今日の日付が表示される場合は，文字の大きさを変えることで，それがわかるようにすること．
  • (B1-2) 百ます計算の表と解答を表示するプログラムを作れ．先頭行の10マス，先頭列の10マスにはそれぞれ1～99の値をランダムに入れる．「加算」，「減算」，「乗算」のラジオボタンを付け，初期値として，加算にチェックを入れておく．また，加算による百ます計算の解答を表示する．「再計算」ボタンを押すと，先頭行と先頭列に改めて1～99の値をランダムに入れ，ラジオボタンで指定した計算法で百ます計算した解答を表示する．

注意

  • レポートには作成したWebページの内容と考察（理解できたこと，工夫したこと）について書き，レポート提出システムで提出すること．
  • 作成したプログラムは，dius01のホームディレクトリにpublic_html/report/R1（ディレクトリ）を作成し，その下に保存すること．ファイル名はそれぞれA1-1.jsp, A1-2.jsp,B1-1.jsp, B1-2.jspとすること．A1-1.jsp, A1-2.jsp,B1-1.jsp, B1-2.jspからリンクされているページのファイル名は任意とする．

R2（クッキーとセッション）

必修

  • (A2-1) A1-1をクッキーを用いて拡張する．初めて利用するユーザに対しては入力画面を表示し，イベント名とその年月日を求めてから，「XX（イベント名）まであとYY日です．」と表示せよ．2回目以降の入力画面には前回入力したイベント名，年月日をデフォルト値として表示すること．
  • (A2-2) セッション情報を用いて簡易電卓を作れ．画面には，数値を表示する欄(x)，数値を入力するボックス(y)，加算・減算・乗算・除算の4つのボタンを表示する．ボタンを押すとx@yを計算し，xに表示する．@は加算・減算・乗算・除算のいずれかである．扱う数値は整数のみで構わない．xの初期値は0とし，表示するだけで，直接入力することはないものとする．

オプション

  • (B2-1) A1-1をクッキーを用いて拡張する．初めて利用するユーザに対しては入力画面を表示し，イベント名とその年月日を求めてから，「XX（イベント名）まであとYY日です．」と表示せよ．2回目以降に利用するユーザに対してはいきなり「XX（イベント名）まであとYY日です．」と表示せよ．一週間後に操作しても正しい日数が表示できるようにすること．
  • (B2-2) 入力画面でメニューの選択と数量の入力（例えば，餃子2個とラーメン1個）ができるようになっており，「注文」ボタンを押すと，注文したメニューの個数と合計金額が加算されていく簡易計算機を作れ．メニューの単価はプログラム中に書き込んでおいてよい（参考：餃子の王将）．初期メニューの個数は3個（例えば，餃子，ラーメン，ライス）とする．また，「リセット」ボタンを押すと，メニューの個数と合計金額を0にする．さらに，「メニュー追加」ボタンを押すと，新しいメニューの名前と単価を入力する画面に遷移し，その入力が終わると新たなメニューが追加された元の入力画面に戻るようにせよ．

注意

  • レポートには作成したプログラムの内容と考察（理解できたこと，工夫したこと）について書き，レポート提出システムで提出すること．
  • 作成したプログラムは，dius01のホームディレクトリにpublic_html/report/R2（ディレクトリ）を作成し，その下に保存すること．ファイル名はA2-1.jsp, A2-2.jsp, B2-1.jsp,B2-2.jspとすること．

R3（クラスの利用）

必修

  • (A3-1) A1-1を拡張し，イベント名入力にHTMLエンコードを施せ．年月日の入力に関しては，文字列やありえない日付が入力された場合は，「入力が正しくありません」とエラー表示するようにせよ．
  • (A3-2) 連想配列に3件以上のイベント名とその年月日を記録しておく．イベント名を入力するとその年月日とその日までの日数を表示するようにせよ．またイベント名が連想配列に含まれていない場合は，その旨表示すること．

オプション

  • (B3-1) 入力フォームにはイベント名，その年月日を入力できるテキストボックスを用意する．また「登録」「検索」ボタンを付加する．「登録」ボタンが押されるとイベント名とその年月日が連想配列に登録される．イベント名を入力して「検索」ボタンが押されると(A3-2)と同様の結果が表示される（登録に必要な年月日は無視される）．連想配列の内容をセッション変数に保存することで，ブラウザが開いている間は，継続して利用可能にすること．HTMLエンコードと入力エラー処理も適切におこなうこと．
  • (B3-2) 月を入力すると，今年のその月のカレンダーを表示するプログラムを作れ．また日曜日は赤字，土曜日は青字で表示すること．さらに，連想配列に祝日を保存し，祝日がカレンダーに含まれる場合は，その日を赤字にし，祝日名を付記すること．

注意

  • レポートには作成したプログラムの内容と考察（理解できたこと，工夫したこと）について書き，レポート提出システムで提出すること．
  • 作成したプログラムは，dius01のホームディレクトリにpublic_html/report/R3（ディレクトリ）を作成し，その下に保存すること．ファイル名はA3-1.jsp, A3-2.jsp, B3-1.jsp, B3-2.jspとすること．

R4（ファイルと正規表現）

必修

  • (A4-1) 5件以上のイベント名とその年月日を「,」で区切って，ファイルに保存しておく．年月日は「2015/11/2」のように「/」で区切って表現すること．イベント中で1件以上は過去，1件以上は未来のイベントとすること．例えば，

    2021/7/23, 東京オリンピック
    2025/4/13, 関西万博

    次に，そのファイルを読み込み，表として表示するプログラムを作成せよ．過去のイベントに関しては文字または背景の色を変えることにより，その違いが分かるようにすること．

  • (A4-2) 正規表現を用いて，イベントを年月日で検索するプログラムを作成せよ．入力画面には年，月，日をそれぞれ入力するテキストボックスを合計3個用意する．年月日を入力するとそれに該当するイベントを表示する．月だけ入力すると（年や日に関わらず）その月のイベントが表示される．年や日も同様に扱う．何も入力せずに検索すると，全てのイベントが表示される．イベントのデータとしては(A4-1)で作成したイベントファイルを用いる．

    ヒント：8月のイベントを検索したい場合は，「.*/8/.*」を正規表現パターンとして年月日を検索すればよい．

  • (A4-3) イベントをファイルに保存するプログラムを作成せよ．入力画面でイベント名とその年月日を入力し，ボタンを押すと，イベント名とその年月日を「,」で区切って，ファイルに追加保存されるようにする．保存するファイル名を(A4-1)と同じにすることにより，入力した結果が(A4-1)のプログラムによって確認できるようにすること．ありえない日付は入力できないようにすること．

オプション

  • (B4-1) (A4-1)を拡張する．2つのボタンを追加し，イベントの表示を年月日の昇順と降順にソートできるようにせよ．
  • (B4-2) (A4-2)を拡張し，イベント名を検索するためのテキストボックスを3個追加する．第1のテキストボックスはイベント名の始まりとマッチする文字列を入力する．第2のテキストボックスはイベント名に含まれる文字列を入力する．第3のテキストボックスはイベント名の終わりとマッチする文字列を入力する．複数のテキストボックスに入力がある場合は，そのすべての条件を満たすイベントを検索する．例えば，第1のテキストボックスに「第90回」，第2のテキストボックスに「野球」，第3のテキストボックスに「大会」が入力された場合は，「第90回選抜高等学校野球大会」のようなイベント名が検索される．

注意:

  • レポートには作成したプログラムの内容と考察（理解できたこと，工夫したこと）について書き，レポート提出システムで提出すること．
  • 作成したプログラムは，dius01のホームディレクトリにpublic_html/report/R4（ディレクトリ）を作成し，その下に保存すること．ファイル名はA4-1.jsp, A4-2.jsp, B4-1.jsp, B4-2.jspとすること．

R5（SQL）

必修

  • (A5-1)イベントデータベースをSQLを用いて作成せよ．登録するイベントは10件以上．その中で1件以上は過去，1件以上は未来のイベントとすること．登録するイベントの属性は，ID番号，イベント名，日付，場所名（都市名），とする．リレーション名はeventsとする．また，以下の質問に答えるSQL文を書け．

    • (a)登録したイベントの情報を全て表示せよ．
    • (b)部分的なイベント名検索を行え．例えば，"オリンピック"で検索すると，"オリンピック"という文字列がイベント名に含まれる全てのイベントが表示される．レポート提出の際には適当な文字列を用いよ．
    • (c)未来のイベントの検索を行え．
    • (d)2022年以外のイベントの検索を行え．（「等しくない」を表す比較演算子は'!='である．）
    • (e)複合検索を行え．例えば，「過去のオリンピック」の検索．クエリ提出の際には適当な条件を用いよ．

  • (A5-2)場所リレーションplacesを作れ．属性としては場所名（都市名），国名，緯度，経度とする．(A5-1)で登録したイベントの場所はすべて，場所リレーションに登録すること．

    • (f)(A5-1)で作成したリレーションeventsと結合して，登録した全てのイベント名，日付，場所，国名を表示せよ．
    • (g)日本で行われるイベントをを日付順にソートせよ．
    • (h)日本で行われるイベントの数を示せ．
    • (i)イベントを神戸三田キャンパス（緯度34.915408，経度135.163968）から近い順にソートせよ．

オプション

  • (B5-1)蔵書データベースをSQLを用いて作成せよ．登録する本は10冊以上とする．まず，本のリレーションbooksを作れ．属性としては，ID番号，著者名，書名，出版社名，ISBN13, 価格，出版年，表紙画像ファイル名，とする． ISBNの形式はハイフン('-')抜きとする．次に，出版社のリレーションpublishersを作れ．属性としては出版社名，住所，郵便番号，電話番号，とする．また，以下の質問に答えるSQL文を書け．
    • (a)登録した本の情報を全て表示せよ．
    • (b)部分的な書名検索を行え．例えば，"Java"で検索すると，"Java"という文字列が書名に含まれる全ての本が表示される．プログラムを提出する際は適当な文字列を用いよ．
    • (c)2500円以上5000円以下という価格検索を行え．
    • (d)出版年がある年（例えば，2000年）でない本の検索を行え．（「等しくない」を表す比較演算子は'!='である．）
    • (e)複合検索を行え．例えば，「著者が北村で，価格が2500円以下の本」．プログラム提出の際には適当な条件を指定せよ．
    • (g)東京の出版社の本を出版年順にソートせよ．
    • (h)大阪で出版された本の数を示せ．
    • (i)複数の出版社から本を出版している著者名を示せ．（表示の重複を避けたい場合は，SELECTの後にDISTINCTを付ける．）

注意:

  • レポートには作成したプログラムの内容（SQL文）と考察（理解できたこと，工夫したこと）について書き，レポート提出システムで提出すること．
  • 作成したプログラム（USE文，SELECT文のみ）はスクリプトファイルの形にし，dius01のホームディレクトリにpublic_html/report/R5（ディレクトリ）を作成し，その下にdb-events.sqlの名前で保存すること． オプション課題に関してはdb-books.sqlの名前で保存すること．
  • 作成すべきプログラムはクエリの部分のみでよい．すなわち，USE文とSELECT文のみ．データはあらかじめINSERT文を用いてデータベースに登録しておくこと．例えば，以下のようなスクリプトファイルを提出する．
USE stuXXXX;
SELECT * FROM events;
以降(b)から(i)までのSELECT文が並ぶ

R6（JDBC）

必修

  • (A6-1)R5で作成したリレーションeventsの内容を表示するプログラムを作成せよ．また，過去のイベントに関しては文字または背景の色を変えることにより，その違いが分かるようにすること．
  • (A6-2)R5で作成したリレーションeventsに，入力画面からデータを追加するプログラムを作成せよ．
  • (A6-3)R5で作成したリレーションeventsに，入力画面を使ってid番号を指定し，該当するデータを削除するプログラムを作成せよ．

オプション

  • (B6-1)R5で作成したリレーションplacesの内容を表示，追加，編集，削除するプログラムを作成せよ．
  • (B6-2)R5で作成したリレーションeventsとplacesを用いて，登録したイベント名，日付，場所，国名で検索できるプログラムを作成せよ．複数の属性値が指定された場合は複合検索すること．例えば，国名と日付が指定された場合は両方を満たすイベントを検索する．
  • (B6-3)(B5-1)で作成した本のデータベースを操作する以下の機能を持つプログラムを作成せよ．
    • (a)データベースに登録されている本の一覧表示．データの属性が何であるのか分かりやすく表示すること．
    • (b)本の追加．入力画面から追加する本のデータを入力する．
    • (c)本の削除．入力画面から削除する本のID番号を入力する．
    • (d)書名検索．書名は入力画面より入力する．
    • (e)拡張書名検索．書名に0以上の任意の長さの文字列を表すワイルドカード（例えば，'*'）を利用可能とする．すなわち，"Java*"は"Java"で始まるタイトル，"*Java"は"Java"で終わるタイトル，"*Java*"は"Java"を含むタイトルを表す．
    • (f)価格検索．ただし価格はキーボードより入力し，該当する価格の本が存在しない場合には，それに最も近い本を出力する．

注意:

  • レポートには作成したプログラムの内容と考察（理解できたこと，工夫したこと）について書き，レポート提出システムで提出すること．
  • 作成したプログラムは，dius01のホームディレクトリにpublic_html/report/R6（ディレクトリ）を作成し，その下に保存すること．ファイル名はA6-1.jsp, A6-2.jsp, A6-3.jspなどとすること．
  • A6-1, A6-2, A6-3を一つのWebページとして作成してもよい．その場合は，その旨をレポートに記載すること．

R7（Webアプリケーション）

必修

  • (A7-1)(A6-1),(A6-2),(A6-3)のプログラムにリンクを張るなどして，一つのトップページ(R7-1.jsp)からデータの表示，追加，削除ができるようにせよ．さらに，以下の機能を追加せよ．これら3つの検索を複合検索にする必要はない．
    • (a)イベント名検索．イベント名を入力するテキストボックスを用意し，該当するイベントを検索する．
    • (b)日付検索．入力画面には年，月，日をそれぞれ入力するテキストボックスを合計3個用意する．年月日を入力するとそれに該当するイベントを表示する．
    • (c)場所検索．場所名を入力するテキストボックスを用意し，該当するイベントを検索する．

オプション

  • (B7-1)（A7-1)で作成したプログラムに以下の機能を加えよ．
    • (d)部分的なイベント名検索を行え．例えば，"オリンピック"で検索すると，"オリンピック"という文字列がイベント名に含まれる全てのイベントが表示される．
    • (e)日付の範囲検索．例えば，2022年4月1日から2023年3月31日までのイベント検索．
    • (f)表eventsの内容をイベント名，日付，場所のいずれにおいても昇順，降順にソーティングして表示できるようにせよ．
  • (B7-2)（イベントと本以外の）データの検索，追加，編集，削除ができるWebアプリケーションを作成せよ．ただし，データの管理にはデータベースを利用すること．

注意:

  • レポートには作成したプログラムの内容と考察（理解できたこと，工夫したこと）について書き，レポート提出システムで提出すること．
  • 作成したプログラムは，dius01のホームディレクトリにpublic_html/report/R7（ディレクトリ）を作成し，その下に保存すること．ファイル名はA7-1.jsp, B7-1.jsp，B7-2.jspなどとすること．
